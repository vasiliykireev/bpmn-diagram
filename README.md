# Описание процесса обработки заказа

## Список процессов

- Прием и оформление заказа от клиента.
- Проверка, есть ли товар в наличии.
- Подтверждение заказа и ожидание оплаты.
- Подготовка заказа к отправке.
- Отгрузка заказа клиенту.
- Обработка ситуации, когда товара нет.


## Ключевые процессы

- Проверка наличия товара: от этого шага зависит, пойдет заказ дальше или будет отменен.
- Подтверждение и оплата: без оплаты заказ не передается на склад.
- Комплектация и отгрузка: финальный этап, который формирует клиентский результат.
- Уведомление клиента: на каждом важном этапе клиент получает понятную информацию.


## Требования к процессу

- Заказ должен быстро приниматься и фиксироваться.
- До подтверждения клиенту обязательно проверяется наличие товара.
- Если товар есть, клиент получает подтверждение и инструкцию по оплате.
- Если товара нет, клиент сразу получает уведомление, а заказ корректно закрывается.
- Сборка и отправка заказа начинаются только после подтвержденной оплаты.
- Ответственность между отделом продаж и складом должна быть четко разделена.


## Этапы процесса

1.	Клиент оформляет заказ.
2.	Отдел продаж принимает и проверяет заказ.
3.	Сотрудник проверяет наличие товара.
4.	Если товар есть:
- клиент получает подтверждение;
- магазин получает оплату оплату;
- склад комплектует и упаковывает заказ;
- заказ отгружается;
- клиент получает уведомление об отгрузке.
5.	Если товара нет:
- клиент получает уведомление об отсутствии товара;
- заказ завершается.
